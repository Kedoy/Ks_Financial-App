# AI Рекомендации — Логика работы

## Обзор

Все рекомендации генерируются **исключительно через ИИ** (OpenRouter API). Жестко закодированных советов нет.

## Структура рекомендаций

Система генерирует рекомендации по строгой структуре:

### 1. **success** (максимум 1) — Хвалебная/мотивирующая
Добавляется **ТОЛЬКО** если пользователь заслуживает похвалы:

**Критерии для похвалы** (хотя бы один):
- ✅ Положительный баланс (доходы > расходов)
- ✅ Сбережения > 20% от дохода
- ✅ Разнообразие категорий (≥3)
- ✅ Регулярные доходы (>1 транзакции)

**Запрет на похвалу** (если хотя бы один):
- ❌ Отрицательный баланс
- ❌ Мало транзакций (<3)
- ❌ "Глупые" траты: рестораны/развлечения > 30% расходов

Если ИИ добавил `success` но пользователь не заслуживает похвалы — рекомендация конвертируется в `info`.

---

### 2. **warning** (1-2) — Негативные/критичные
Указывают на явные проблемы:
- Отрицательный баланс
- Одна категория > 50% расходов
- Мало транзакций для анализа
- Отсутствие доходов

Будь прямым и конкретным, используешь цифры.

---

### 3. **info** (1-2) — Практические советы по экономии
Конкретные действия с измеримой выгодой:
- Что именно сделать
- Сколько денег это сэкономит
- Избегаем общих фраз без конкретики

**Пример правильного info:**
> "На рестораны потрачено 15000 ₽ (45% расходов). Сократите до 2 раз в неделю — сэкономите ~8000 ₽/мес"

**Примеры запрещённых советов:**
- ❌ "Разбейте расходы на меньшие суммы" — не экономит деньги
- ❌ "Следите за тратами" — без конкретики
- ❌ "Планируйте бюджет" — без конкретных шагов

## Технические детали

### API Endpoint
```
GET /api/v1/analytics/ai-insights/?days=30
```

### Модель
- **OpenRouter**: `arcee-ai/trinity-mini:free` (быстрая и надёжная)
- **Temperature**: 0.7
- **Max tokens**: 1500

### Кэширование
- Успешные ответы: **1 час**
- Ошибки: **5 минут**

### Файлы
- `backend/apps/core/services/openrouter_service.py` — сервис ИИ
- `backend/apps/analytics/views.py` — Django view
- `my-finance-app/src/components/AIInsights.jsx` — фронтенд компонент

---

## Алгоритм работы

1. **Сбор данных**: Получение транзакций пользователя за период
2. **Анализ**: Расчёт метрик (баланс, доли категорий, средние значения)
3. **Генерация промпта**: Формирование детального запроса для ИИ
4. **Запрос к API**: Отправка данных в OpenRouter
5. **Парсинг ответа**: Извлечение JSON с рекомендациями
6. **Нормализация**:
   - Проверка можно ли хвалить пользователя
   - Конвертация `success` → `info` если нельзя хвалить
   - Ограничение количества рекомендаций каждого типа
7. **Кэширование**: Сохранение результата
8. **Возврат**: Отправка клиенту

---

## Методы нормализации

### `_can_praise_user(data)`
Проверяет заслуживает ли пользователь похвалы.

### `_normalize_insights(insights, data)`
Применяет строгую структуру:
- Максимум 1 `success` (только если можно хвалить)
- 1-2 `warning`
- 1-2 `info`

---

## Обработка ошибок

### Если API недоступен
- Возвращается пустой список рекомендаций
- Фронтенд показывает "AI-сервис временно недоступен"

### Если нет транзакций
- Возвращается `info`: "Добавьте транзакции для получения рекомендаций"

### Rate limit (429)
- Кэшируется ошибка на 5 минут
- Пользователь видит "Превышен лимит запросов"
