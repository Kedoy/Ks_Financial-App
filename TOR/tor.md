2. Функциональные Требования
2.1. Система Аккаунтов и Аутентификации

    Регистрация/Вход: Пользователь должен иметь возможность зарегистрироваться и войти с использованием email/пароля.

    Безопасность: Использование токенов (например, JWT) для защищенного доступа к API.

    Восстановление пароля: Предусмотреть механизм восстановления пароля через email.

2.2. Настройка и Конфигурация

    Указание Номера Банка: В настройках пользователь указывает номер телефона банка, с которого приходят SMS о транзакциях. Приложение должно отслеживать SMS только с этого номера.

    Разрешения: Приложение должно запросить и получить разрешение на чтение SMS-сообщений.

2.3. Обработка SMS (Парсинг)

    Получение SMS: Приложение должно перехватывать входящие SMS с указанного в настройках номера.

    Алгоритм Парсинга: Разработка надежного парсера для извлечения следующих данных:

        Сумма транзакции.

        Тип транзакции (Списание/Пополнение).

        Дата и время.

        Описание/Назначение (например, "Покупка в магазине X").

    Фильтрация: Игнорирование всех SMS, которые не соответствуют паттерну транзакции.

2.4. Взаимодействие с Бэкендом (API)

    Отправка Данных: После успешного парсинга данные о транзакции отправляются на сервер через защищенный API-эндпоинт (POST /api/transactions/upload).

    Получение Истории: Приложение должно запрашивать и отображать полную историю расходов пользователя (GET /api/transactions/history).

    Синхронизация: Обеспечить механизм синхронизации данных между клиентом и сервером.

2.5. Пользовательский Интерфейс

    Главный Экран: Отображение списка последних транзакций.

    Экран Аналитики:

        Сводка: Общая сумма расходов/доходов за период (день, неделя, месяц).

        Графики: Визуализация расходов по категориям (круговая диаграмма) и динамика расходов во времени (линейный/столбчатый график).
