# Финансовый Помощник (Мобильное приложение)

## 1. Работа приложения
### 1.1 Обработка сообщений
*Предполагается, что пользователь уже авторизован и настроил номер банка.*  
Приложение слушает BANK_PHONE, парсит данные из смс в новый расход и сохраняет в БД:
- Дата
- Сумма
- Категория: неотсортированные

Так как смс банка не указывает категорию расхода, это необходимо делать пользователю вручную.  

**Желаемое**: предлагать пользователю назначить категорию через уведомление (как было в лабораторной работе с изучением английского языка)  
**Альтернатива** *(в крайнем случае)*: пользователь открывает приложение, ему предлагают в всплывающем окне выбрать категории для неотсортированных расходов.
### 1.2 Работа с API
Все изменения, связанные с расходами, должны отправляться на бекенд.   
Единственное хранилище данных - сервер.  
см. [API](#3-api-эндпоинты)


## 2. Визуальная часть: экраны
### 2.1 Вход/регистрация пользователя
*Первое, что видит пользователь. Для продолжения работы необходима авторизация.*  
[Данные отправляются по API](#31-регистрация-нового-пользователя). Необходимые поля ввода:
- Для входа: имя аккаунта, пароль
- Для регистрации: имя аккаунта, пароль, подтверждение пароля
### 2.2 Главный экран - расходы и аналитика
*[Источник данных](#33-получить-данные-о-расходах-пользователя)*  
Одна карточка - один день.
Общий вид:
```
    12 Декабря 2025  + Добавить (день)
    ===============================================  
    |                       + Добавить (категорию)|
    |  Категория 1  (скрыть)                      |  
    |     |- Расход1         100р                 |  
    |     |- Расход2         200р                 |  
    |     |- + Добавить (расход)                  |
<<< |                                             |  >>>
    |  Категория 2  (раскрыть)                    |  
    |                                             |  
    ===============================================  
```
Реализовать функционал:
- Добавлять новые дни
- Создавать категории, расходы
- Удалять дни, категории, расходы
- Редактировать имя категории, имя и сумму расхода  

Аналитика:
- Диаграмма-пирог со статистикой сумм расходов по категориям за день
- Гистограмма с суммами расходов по дням (за месяц)

### 2.3 Экран настроек
    Возможность указать номер телефона, смс которого будет парситься
